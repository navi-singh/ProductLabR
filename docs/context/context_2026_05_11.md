## What the product is

ProductLabR is an **editorial product-review site** — long-form, ranked "best of" buying guides for consumer electronics and outdoor gear. It is content-first (every review is a markdown file), built on Next.js App Router with full static export to GitHub Pages, and monetized through Google AdSense display ads and affiliate links.

**Core value proposition:** Help shoppers decide *which* product to buy in a specific category, then earn revenue when they click out to retailers or view ads on the way.

## Tech stack

| Layer | Choice |
|---|---|
| Framework | Next.js 16, App Router, React 19, TypeScript 5 strict |
| Styling | Tailwind CSS 3.4 + CSS variables (`styles/theme.css`) |
| Content | Markdown via `marked` + YAML frontmatter via `gray-matter` |
| Motion | Framer Motion (used sparingly) |
| Analytics | `@next/third-parties` (Google Analytics + AdSense) |
| Hosting | GitHub Pages (static export to `./out/`) |
| CI/CD | GitHub Actions (`.github/workflows/nextjs.yml`) |

**Critical architectural constraint:** Build-time static rendering only. No server runtime in production. No ISR, no edge functions. Every dynamic route must enumerate its params via `generateStaticParams()`. Personalization must run client-side.

## Data layer

**Two-tier:**
1. `lib/Posts.ts` — generic markdown loader, wrapped in `React.cache()`. Reads `posts/<category>/<slug>.md`, validates slugs, parses frontmatter.
2. Vertical-specific helpers — `lib/power-station-data.ts` exposes typed `PowerStationEntry[]` with filter functions: `getStationsByFeature()`, `getStationsUnderPrice()`, `getStationsBySlugs()`, `getQuickPicks()`. New verticals should follow this pattern.

## Design system

### Type scale — 5 roles (enforced, no arbitrary sizes)

| Class | Font | Size / Weight | Use |
|---|---|---|---|
| `.type-display` | Playfair Display | 32px / 800 | Hero `<h1>`, article titles |
| `.type-headline` | Playfair Display | 20px / 700 | Section headings, card H2s |
| `.type-title` | Inter | 15px / 600 | Product names in cards |
| `.type-body` | Inter | 14px / 400 | Descriptions, summaries, prose |
| `.type-label` | Inter | 11px / 600 + uppercase | Badges, metadata, dates |

### Color tokens (key roles)

| Token | Value | Use |
|---|---|---|
| `--primary` | `#007ACC` | Header bg, score badges 8.0–8.9, active nav |
| `--accent` | `#E87B35` | Score badges ≥9.0, primary CTAs, award badges, TrustBar checks |
| `--primary-lightest` | `#E6F2FF` | Card hover tint, selected chip bg |
| `--success` | semantic green | "In Stock" only — not score badges |

**Score badge rule:** ≥9.0 = accent orange, 8.x = primary blue, 7.x = amber, <7 = neutral gray.

### State layers (MD3 interaction model)
`.state-layer` and `.state-layer-light` in `styles/global.css` provide hover/active overlays on interactive cards. Cards also get `hover:-translate-y-0.5 hover:shadow-card-hover` for lift. Note: `overflow-hidden` is baked in — child `box-shadow` and overflow dropdowns will clip.

## Navigation

**Desktop (md+):** Sticky `<Header>` — logo, `NAV_LINKS`, `CategoryDropdown`, `SearchBar` (disabled).

**Mobile (< md):** `<MobileBottomNav>` — fixed bottom bar (Home · Best Of · Search · Categories) + slide-up category sheet. Both consume `NAV_CATEGORIES` from `lib/nav-categories.ts`. Adding a category there auto-populates both surfaces.

`<main>` in `layout.tsx` has `pb-16 md:pb-0` to clear the mobile bottom bar.

## Monetization

**AdSense display ads:**
- All slots defined in `lib/adsense-config.ts`
- `<AdBanner adSlot="..." adFormat="..." />` renders a labeled gray placeholder in dev; real ads in prod when `NEXT_PUBLIC_ADSENSE_PUBLISHER_ID` is set
- Build warns (not errors) if the env var is missing — do not tighten this back to an error

**Affiliate links:**
- Defined in frontmatter `retailerLinks` map
- All URLs pass `isSafeUrl()` before render
- All outbound links get `rel="noopener noreferrer nofollow"`

**Revenue expectations:** $0–$50 month 1, $50–$200 month 3, $200+ month 6+.

## Content schema (key rules)

Every review is `posts/<category>/<slug>.md` with YAML frontmatter + markdown body.

**Required frontmatter:** `title`, `date`, `price`, `productImage`, `specs`, `pros`, `cons`, `retailerLinks`, `ratingBreakdown.metrics[]`

**Scoring:** Overall score = arithmetic mean of `ratingBreakdown.metrics[].score` (0–10 scale). Stars = `score / 2`.

**Price visibility rule:** Prices rendered on individual article pages only. Never shown on cards (ReviewCard, RankedProductCard, Best Of guide cards, hero). Cards will eventually carry "Check Price →" affiliate links.

**Power station extras:** `capacityWh` (number) and `features` (string[] of tags: `30a-rv`, `solar`, `solar-kit`, `240v`, `cpap`, `van-life`) required for sub-category page filtering.

**Slug rules:** Lowercase, `_` or `-` separators, matches `/^[a-z0-9]+(?:[_-][a-z0-9]+)*$/i`. Renaming after publish breaks inbound links.

## Build & deploy

```bash
npm ci                  # install
npm run dev             # localhost:3000
npm run test:build      # lint + type-check + build (the only CI gate)
npm run build:analyze   # bundle analysis
npm run preview         # build + serve static output
```

CI pipeline (push to `main`): install → build → upload `./out/` → deploy to GitHub Pages. Typical duration: 2–3 minutes.

**Performance targets:** LCP < 2.5s, CLS < 0.1, FID < 100ms. Lighthouse: Performance > 90, Accessibility > 95, SEO > 95.

## Conventions

- **TypeScript strict.** `interface` over `type`. No `enum` — use string literal unions.
- **Default RSC.** Add `'use client'` only for state/effects/event handlers.
- **No `useEffect` for data.** Fetch in server components; pass props down.
- **Images:** always `OptimizedImage`. Never raw `<img>`.
- **External URLs:** always `isSafeUrl()` before rendering an href.
- **Class merging:** `cn()` (clsx + tailwind-merge).
- **Variants:** `class-variance-authority` for ≥3 visual variants.
- **Comments:** only for non-obvious *why*. No *what* explanations.
- **No emojis** in code or comments unless explicitly requested.

## Known constraints & gaps

| Constraint | Impact |
|---|---|
| Static-only GitHub Pages hosting | No personalization, no A/B testing, no server-side rendering |
| No automated test suite | Regressions caught only via `npm run test:build` (lint + type-check + build). 80% coverage is a known target gap. |
| AdSense `'unsafe-inline'` CSP | Weaker than ideal; acceptable for revenue trade-off |
| `marked` (not MDX) | No React components inside content — intentional, keeps content authorable by non-devs |
| Image domains whitelisted in `next.config.mjs` | New CDNs need a code change + redeploy |
| AdSense build warning | Missing `NEXT_PUBLIC_ADSENSE_PUBLISHER_ID` emits a warning — do NOT tighten to an error |
